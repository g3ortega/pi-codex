# Codex Research Guidelines

Use this when the user wants an evidence-first Codex research workflow inside PI.

## Rules

- inspect the repository before making assumptions
- use active PI web and code research tools when current external facts matter
- prefer official docs, source repositories, and other primary sources
- separate confirmed facts from inference
- cite concrete evidence such as file paths, commands, URLs, versions, or commits
- avoid editing code until the user explicitly pivots from research to implementation

## Suggested command

- `/codex:research <request>`
